Introduction to Machine Learning & Data Preprocessing

What is Machine Learning?

Arthur Samuel (1959): "Field of study that gives computers the ability to learn without being explicitly programmed."

Traditional Programming: Data + Program → Output
Machine Learning: Data + Output → Program (Model)

Types of ML Algorithms

Supervised Learning
Input x and correct output y are given.
Used for: Regression & Classification
  • Regression: Predict continuous value (e.g. house price)
  • Classification: Predict discrete class (e.g. tumor: malignant/benign)
Unsupervised Learning
Only input x, no output labels y.
Algorithm finds structure in data.
Used for: Clustering
  • Group news articles by topic
  • Discover market segments
⚠️ Key Exam Trick: Predicting how many items sell = Regression. Detecting hacked accounts = Classification. Grouping customers automatically = Unsupervised (Clustering).

Attribute Types

TypeDescriptionExample
NominalNo order, categories onlyHair color, zip codes, ID
BinarySpecial nominal, 2 states (0/1)Gender (symmetric), HIV test (asymmetric)
OrdinalOrdered, but unknown magnitude between valuesSize: small/medium/large, grades
IntervalOrdered, equal units, NO true zeroTemperature in °C, calendar dates
RatioOrdered, equal units, HAS true zeroTemperature in Kelvin, length, weight, counts
⚠️ Interval vs Ratio: Temperature in °C is Interval (0°C is not "no temperature"). Temperature in Kelvin is Ratio (0K = absolute zero = true zero).

Discrete: Finite/countably infinite values (zip codes, words).
Continuous: Real-valued (temperature, height, weight).

Distance Measures

Euclidean (L2): d(i,j) = √(Σ(xik - xjk)²)
Manhattan (L1): d(i,j) = Σ|xik - xjk|
Dissimilarity (Binary): d(i,j) = (r + s) / (q + r + s + t)

where q=both 1, r=i=1 j=0, s=i=0 j=1, t=both 0 — asymmetric attributes ignore t (both 0).

Data Preprocessing

Handling Missing Values:

  • Ignore the tuple (if class label missing)
  • Fill manually
  • Fill with attribute mean, class mean, or most probable value

Binning (Noise Handling):

Equal-Width: W = (Max − Min) / N — each bin has equal range.
Equal-Depth: Each bin has equal number of values = Total / N bins.

Smoothing methods: by bin mean, by bin median, by bin boundaries.

Example: values 4,8,9,15,21,21,24,25,26,28,29,34 — 3 bins
Equal-Depth (4 per bin): [4,8,9,15] | [21,21,24,25] | [26,28,29,34]
Smooth by bin mean: [9,9,9,9] | [23,23,23,23] | [29,29,29,29]
Smooth by boundaries: [4,4,4,15] | [21,21,25,25] | [26,26,26,34]

Normalization

Min-Max Normalization:

v' = (v − min_A) / (max_A − min_A) × (new_max − new_min) + new_min

Example: v=73,600; min=12,000; max=98,000 → mapped to [0,1]:

v' = (73600−12000)/(98000−12000) × 1.0 = 61600/86000 = 0.716

Z-Score Normalization:

v' = (v − μ_A) / σ_A

Example: μ=54,000; σ=16,000; v=73,600 → v' = (73600−54000)/16000 = 1.225

Decimal Scaling:

v' = v / 10^j where j = smallest integer such that max(|v'|) < 1

Data Reduction Strategies

  • Dimensionality reduction: PCA, Wavelet transforms, Feature subset selection
  • Numerosity reduction: Regression, histograms, clustering, sampling
  • Data compression

Discretization Methods

Converts continuous attributes to intervals/categories: Binning, Histogram analysis, Clustering analysis, Decision-tree analysis, Correlation analysis (χ²).

Linear Regression (One & Multiple Variables)

Model Representation

Notation: m = number of training examples, x = features (input), y = target output, (x⁽ⁱ⁾, y⁽ⁱ⁾) = i-th training example.

Hypothesis: h_θ(x) = θ₀ + θ₁x

θ₀ = y-intercept (bias), θ₁ = slope (weight)

Cost Function (Mean Squared Error)

J(θ₀, θ₁) = 1/(2m) × Σᵢ₌₁ᵐ (h_θ(x⁽ⁱ⁾) − y⁽ⁱ⁾)²

Goal: Find θ₀, θ₁ that minimize J(θ₀, θ₁)

Gradient Descent Algorithm

Iteratively update parameters until convergence. Must update simultaneously!

θⱼ := θⱼ − α × (∂J/∂θⱼ) [simultaneously for all j]

For linear regression:

∂J/∂θ₀ = (1/m) × Σ(h_θ(x⁽ⁱ⁾) − y⁽ⁱ⁾)
∂J/∂θ₁ = (1/m) × Σ(h_θ(x⁽ⁱ⁾) − y⁽ⁱ⁾) × x⁽ⁱ⁾
⚠️ Learning Rate α:
Too small → converges very slowly.
Too large → may diverge (J oscillates or increases).
Try: 0.001, 0.003, 0.01, 0.03, 0.1, 0.3, 1 (multiply by ~3 each step).

Multiple Variables

h_θ(x) = θ₀ + θ₁x₁ + θ₂x₂ + ... + θₙxₙ = θᵀx

(Define x₀ = 1, so θᵀx = Σⱼ₌₀ⁿ θⱼxⱼ)

∂J/∂θⱼ = (1/m) × Σ(h_θ(x⁽ⁱ⁾) − y⁽ⁱ⁾) × xⱼ⁽ⁱ⁾

Feature Scaling

Make features on similar scale so gradient descent converges faster.

x_scaled = x / max(x) OR x_scaled = (x − μ) / (max − min)

Target: approximately −1 ≤ xᵢ ≤ 1

Feature Engineering & Polynomial Regression

Create new features by combining existing ones. Example: Area = frontage × depth → use as new feature x₃.

h_θ(x) = θ₀ + θ₁x + θ₂x² + θ₃x³ (Polynomial Regression)
📌 Worked Example (from lecture): x=[2,3,4], y=[3,4,5], θ₀=2, θ₁=2, α=0.1

h(2)=6, h(3)=8, h(4)=10  |  errors: (6-3)=3, (8-4)=4, (10-5)=5
∂J/∂θ₀ = 1/3×(3+4+5) = 12/3 = 4
∂J/∂θ₁ = 1/3×(3×2+4×3+5×4) = 1/3×(6+12+20) = 38/3 ≈ 12.67
θ₀_new = 2 − 0.1×4 = 1.6  |  θ₁_new = 2 − 0.1×12.67 ≈ 0.733

Logistic Regression (Classification)

Why Not Linear Regression for Classification?

Linear regression can predict values >1 or <0, which makes no sense for probabilities. We need h(x) ∈ [0,1].

Sigmoid (Logistic) Function

g(z) = 1 / (1 + e^(−z))
h_θ(x) = g(θᵀx) = 1 / (1 + e^(−θᵀx))

Interpretation: h_θ(x) = P(y=1 | x; θ) — probability that y=1 given x.

Sigmoid properties:
z → +∞: g(z) → 1  |  z → −∞: g(z) → 0  |  z = 0: g(z) = 0.5

Decision Boundary

Predict y=1 if h_θ(x) ≥ 0.5, which means θᵀx ≥ 0.
Predict y=0 if h_θ(x) < 0.5, which means θᵀx < 0.

Decision boundary: θᵀx = 0 (i.e., z = 0)

Example: θ = [−3, 1, 1] → decision boundary: −3 + x₁ + x₂ = 0 → x₁ + x₂ = 3

Cost Function for Logistic Regression

Cannot use MSE because it becomes non-convex with sigmoid. Instead:

Cost(h_θ(x), y) = −log(h_θ(x)) if y = 1
Cost(h_θ(x), y) = −log(1 − h_θ(x)) if y = 0

Combined (simplified):

J(θ) = −(1/m) × Σ[y⁽ⁱ⁾·log(h_θ(x⁽ⁱ⁾)) + (1−y⁽ⁱ⁾)·log(1−h_θ(x⁽ⁱ⁾))]

Gradient Descent for Logistic Regression

θⱼ := θⱼ − (α/m) × Σ(h_θ(x⁽ⁱ⁾) − y⁽ⁱ⁾) × xⱼ⁽ⁱ⁾
⚠️ The update rule LOOKS identical to linear regression, but h_θ(x) is now the sigmoid function, not θᵀx directly!

Multiclass Classification

y ∈ {0,1,2,...,K}: Use One-vs-All (OvA) — train K binary classifiers, predict class with highest probability.

📌 Worked Example: x=[3,5,10,12], y=[1,1,0,0], θ₀=2, θ₁=1
z(x) = 2 + 1·x
h(3) = 1/(1+e⁻⁵) ≈ 0.993, h(5) = 1/(1+e⁻⁷) ≈ 0.999
h(10) = 1/(1+e⁻¹²) ≈ 1.0, h(12) = 1/(1+e⁻¹⁴) ≈ 1.0
Decision boundary: 2 + x = 0 → x = −2 (all x>0 classified as 1 → misclassifying x=10,12)

Confusion Matrix (Binary Classification)

Predicted PositivePredicted Negative
Actual PositiveTP (True Positive)FN (False Negative)
Actual NegativeFP (False Positive)TN (True Negative)

Classification Metrics

Accuracy = (TP + TN) / (TP + FP + FN + TN)
Precision = TP / (TP + FP) ← Of all predicted positive, how many are actually positive?
Recall = TP / (TP + FN) ← Of all actual positive, how many did we find?
F1-Score = 2 × (Precision × Recall) / (Precision + Recall) ← Harmonic mean
📌 Worked Example: 200 emails, Spam/Not Spam
TP=120 (predicted spam, actually spam), FP=20 (predicted spam, actually not spam)
FN=10 (predicted not spam, actually spam), TN=50 (predicted not spam, actually not spam)

Accuracy = (120+50)/200 = 170/200 = 85%
Precision = 120/(120+20) = 120/140 = 85.7%
Recall = 120/(120+10) = 120/130 = 92.3%
F1 = 2×(0.857×0.923)/(0.857+0.923) = 88.9%
⚠️ Precision vs Recall trade-off: In medical diagnosis, high Recall is critical (don't miss sick patients). In spam filtering, high Precision may be preferred (don't delete real emails). F1 balances both.

Overfitting, Underfitting & Regularization

Bias-Variance Tradeoff

Underfitting (High Bias)
Model too simple → high training error AND high test error.
Cannot capture underlying pattern.
Overfitting (High Variance)
Model too complex → low training error BUT high test error.
Memorizes training data, fails to generalize.

Good fit: Low training error AND low test error (generalizes well).

Addressing Overfitting

  1. Collect more training data — most effective solution
  2. Feature Selection — remove irrelevant features (risk: losing useful features)
  3. Regularization — penalize large parameter values
  4. Cross-Validation

Regularization

Ridge Regularization (L2):

J_ridge(θ) = (1/2m) × Σ(h(x⁽ⁱ⁾)−y⁽ⁱ⁾)² + λ × Σⱼ₌₁ⁿ θⱼ²

Shrinks all weights toward zero but never to exactly zero. Good for dense solutions.

Lasso Regularization (L1):

J_lasso(θ) = (1/2m) × Σ(h(x⁽ⁱ⁾)−y⁽ⁱ⁾)² + λ × Σⱼ₌₁ⁿ |θⱼ|

Can shrink weights to exactly zero → performs feature selection automatically.

⚠️ λ (lambda) is the regularization parameter:
λ too large → underfitting (all θ → 0)
λ = 0 → no regularization (may overfit)
Choose λ using cross-validation.

Regularized Gradient Descent:

θⱼ := θⱼ(1 − α·λ/m) − (α/m)·Σ(h(x⁽ⁱ⁾)−y⁽ⁱ⁾)·xⱼ⁽ⁱ⁾ [j ≥ 1]

(θ₀ is not regularized)

K-Fold Cross Validation

Splits data into K equal folds. Each fold is used once as validation set, K−1 folds for training. Repeat K times, average the results.

Recommended range: 5 ≤ K ≤ 10

K=10 is most common. When K=m (number of samples) → Leave-One-Out Cross Validation (LOOCV).

K-Nearest Neighbors (KNN) & Naïve Bayes

KNN Algorithm

Non-parametric, instance-based learning. Can be used for both classification and regression.

Steps:

  1. Calculate Euclidean distance from test point to all training points
  2. Rank all points by distance (ascending)
  3. Select K nearest neighbors
  4. For Classification: take majority class vote
  5. For Regression: take average of K neighbors' values
Euclidean Distance: d = √((x₂−x₁)² + (y₂−y₁)²)
📌 KNN Classification Example (K=3):
Given points with distances: 2(class A), 3(class A), 4(class B), 5(class A), 6(class B)
3 nearest: A, A, B → Majority = A
📌 KNN Regression Example (K=3): neighbors with weight values 1, 2, 3
Prediction = (1+2+3)/3 = 2

Choosing K

  • Small K: sensitive to noise (overfitting)
  • Large K: smoother boundary (may underfit)
  • Common practice: use cross-validation to find optimal K
⚠️ KNN is a stable classifier → Bagging does NOT improve it much.

Naïve Bayes Classifier

Assumption: Features are conditionally independent given the class (the "naïve" assumption).

P(class | x) ∝ P(class) × Π P(xᵢ | class)

Steps:

  1. Calculate prior P(class) for each class
  2. Calculate likelihood P(feature | class) for each feature value
  3. Multiply: posterior ∝ prior × product of likelihoods
  4. Predict class with highest posterior probability
📌 Example: New instance to classify whether to "Play Tennis":
P(Yes|new) = P(Yes) × P(Outlook=Sunny|Yes) × P(Wind=Strong|Yes) × ...
P(No|new) = P(No) × P(Outlook=Sunny|No) × P(Wind=Strong|No) × ...
Predict the class with higher posterior.

Laplace Smoothing: Add 1 to each count to avoid zero probability for unseen features.

Ensemble Methods

What is Ensemble Learning?

Combine multiple diverse models to make better predictions than any single model. Like asking 50 people instead of 1 — diverse group makes better decisions.

Weak learner + Weak learner + ... = Strong learner

Benefits: Better Accuracy Avoids Overfitting Higher Confidence

Categorization

Homogenous Ensemble
Same learning algorithm, different training data.
Methods: Bagging, Boosting
Heterogeneous Ensemble
Different learning algorithms.
Methods: Voting, Stacking

1. Voting

  • Max Voting: Each model votes → take the majority class
  • Averaging: Average all predictions (regression or probability)
  • Weighted Averaging: Weight better models more

Training: Parallel. Data: Complete training set. Different classifiers.

2. Bagging (Bootstrap Aggregating)

Sample training data with replacement (bootstrap sampling) → train same algorithm on each subset → combine by majority vote (classification) or average (regression).

Goal: Reduce VARIANCE (fight overfitting)

Training: Parallel. Example: Random Forest, Bagged Decision Trees, Extra Trees

⚠️ Bagging works only for unstable classifiers (decision trees, neural nets, linear regression). KNN is stable → bagging won't help.

3. Boosting

Models trained sequentially. Each model focuses on misclassified examples from previous model.

Goal: Reduce BIAS (fight underfitting)

Process:

  1. Initialize equal weights for all samples
  2. Train model → find misclassified examples
  3. Increase weights of misclassified, decrease correctly classified
  4. Train next model with new weights → repeat
  5. Final prediction: weighted majority vote (α = importance based on accuracy)
α = ln((1−error)/error) [importance of model]
Update weight (correct): w × e^(−α)
Update weight (wrong): w × e^(+α)

Examples: AdaBoost Gradient Boosting XGBoost LightGBM

4. Stacking

Train multiple base models → their predictions become features for a second-level (meta) model.

Steps: (1) Split into K folds. (2) Train base models on K-1 folds. (3) Predict on Kth fold. (4) Use these predictions as features for meta-model. (5) Meta-model makes final prediction.

Comparison Table

MethodSamplingTrainingGoalCombination
VotingComplete setParallelImprove predictionMajority/Average
BaggingRandom with replacementParallelReduce VarianceAverage/Majority
BoostingWeighted resamplingSequentialReduce BiasWeighted majority vote
StackingK-foldSequentialCombine strengthsMeta-model
📌 Boosting Worked Example: 4 samples, α=0.55 (error=0.25)
Correct samples: multiply weight by e^(−0.55) = 0.577 → 0.25×0.577=0.145
Wrong samples: multiply weight by e^(+0.55) = 1.733 → 0.25×1.733=0.433
Normalize so all weights sum to 1.

Support Vector Machines (SVM)

What is SVM?

A supervised learning algorithm that finds the optimal hyperplane that maximizes the margin between classes. Used for both classification and regression.

Key Concepts

  • Hyperplane: Decision boundary that separates classes. In 2D = line, in 3D = plane, in nD = hyperplane.
  • Support Vectors: The training examples closest to the decision boundary. These are the critical points — remove the rest and the boundary doesn't change.
  • Margin: Distance between the hyperplane and the nearest support vector on each side. SVM maximizes this margin.
Margin = 2 / ||w|| where w is the weight vector
Maximize margin ↔ Minimize (1/2)||w||²

Linearly Separable Problems (Hard Margin SVM)

When data can be perfectly separated by a straight line/hyperplane.

Constraint: yᵢ(w·xᵢ + b) ≥ 1 for all training examples i

The decision boundary satisfies: w·x + b = 0
Positive class boundary: w·x + b = +1
Negative class boundary: w·x + b = −1

Functional margin: H1: wᵀx + b ≥ +1 for y=+1, H2: wᵀx + b ≤ −1 for y=−1

Linearly Non-Separable Problems (Soft Margin & Kernels)

When data cannot be perfectly separated, we use:

Soft Margin (Slack Variables ξ):

yᵢ(w·xᵢ + b) ≥ 1 − ξᵢ (ξᵢ ≥ 0)
Minimize: (1/2)||w||² + C·Σξᵢ

C = regularization parameter: large C → small margin (may overfit), small C → large margin (may underfit)

Kernel Trick:

Map data to higher-dimensional space where it becomes linearly separable, without computing the transformation explicitly.

KernelFormulaUse Case
LinearK(x,z) = xᵀzLinearly separable data
PolynomialK(x,z) = (xᵀz + c)^dNon-linear, polynomial boundary
RBF (Gaussian)K(x,z) = exp(−||x−z||²/2σ²)Most common, non-linear
SigmoidK(x,z) = tanh(αxᵀz + c)Neural network-like boundary

SVM for Regression (SVR)

Support Vector Regression — fits the data within an ε-tube (epsilon-insensitive zone). Points inside the tube contribute zero loss; only points outside are penalized.

SVM vs Logistic Regression

SVM:
  • Maximizes geometric margin
  • Only support vectors matter
  • Works well for high-dimensional data
  • Kernel trick for non-linear data
Logistic Regression:
  • Maximizes likelihood
  • All examples contribute
  • Outputs probabilities directly
  • Faster training, interpretable

SOLUTION — Problem 1: Linear Regression Gradient Descent (8 marks)

(a) Hypothesis function:

h_θ(x) = θ₀ + θ₁x = 0 + 0·x = 0 (initially, before any iterations)

(b) Calculate h_θ(x) for each example:

m = 3 training examples, θ₀=0, θ₁=0
h(1) = 0 + 0(1) = 0   |   error = (0 − 2) = −2
h(2) = 0 + 0(2) = 0   |   error = (0 − 4) = −4
h(3) = 0 + 0(3) = 0   |   error = (0 − 6) = −6

(c) Cost Function J(θ):

J(θ) = 1/(2m) × Σ(h(xᵢ) − yᵢ)²
J = 1/(2×3) × [(−2)² + (−4)² + (−6)²]
J = 1/6 × [4 + 16 + 36]
J = 56/6 ≈ 9.333

(d) Gradients:

∂J/∂θ₀ = (1/m) × Σ(h(xᵢ) − yᵢ)
= (1/3) × [(−2) + (−4) + (−6)]
= (1/3) × (−12) = −4
∂J/∂θ₁ = (1/m) × Σ(h(xᵢ) − yᵢ)·xᵢ
= (1/3) × [(−2)(1) + (−4)(2) + (−6)(3)]
= (1/3) × [−2 − 8 − 18]
= (1/3) × (−28) = −28/3 ≈ −9.333

(e) Updated Parameters:

θ₀_new = θ₀ − α × (∂J/∂θ₀) = 0 − 0.1 × (−4) = 0 + 0.4 = 0.4
θ₁_new = θ₁ − α × (∂J/∂θ₁) = 0 − 0.1 × (−9.333) = 0 + 0.933 = 0.933

The new hypothesis after iteration 1: h(x) = 0.4 + 0.933x

SOLUTION — Problem 2: Logistic Regression (8 marks)

(a) Calculate z = −3 + 1·x:

x=1: z = −3 + 1 = −2   |   x=2: z = −3 + 2 = −1
x=3: z = −3 + 3 = 0   |   x=4: z = −3 + 4 = +1

(b) Sigmoid h(x) = 1/(1+e^(−z)):

x=1 (y=0): h = 1/(1+e²) = 1/(1+7.389) = 1/8.389 ≈ 0.119
x=2 (y=0): h = 1/(1+e¹) = 1/(1+2.718) = 1/3.718 ≈ 0.269
x=3 (y=1): h = 1/(1+e⁰) = 1/(1+1) = 1/2 = 0.500
x=4 (y=1): h = 1/(1+e⁻¹)= 1/(1+0.368)= 1/1.368 ≈ 0.731

(c) Decision Boundary:

z = 0 → −3 + 1·x = 0 → x = 3

Decision boundary is at x = 3: predict y=1 if x ≥ 3, predict y=0 if x < 3.

(d) Gradients and Parameter Update:

Errors (h−y): (0.119−0)=0.119, (0.269−0)=0.269, (0.500−1)=−0.500, (0.731−1)=−0.269
∂J/∂θ₀ = (1/4)×(0.119+0.269−0.500−0.269) = (1/4)×(−0.381) = −0.095
∂J/∂θ₁ = (1/4)×[0.119×1 + 0.269×2 + (−0.500)×3 + (−0.269)×4]
= (1/4)×[0.119 + 0.538 − 1.500 − 1.076]
= (1/4)×(−1.919) = −0.480
θ₀_new = −3 − 0.1×(−0.095) = −3 + 0.0095 = −2.9905 ≈ −2.990
θ₁_new = 1 − 0.1×(−0.480) = 1 + 0.048 = 1.048
SOLUTION — Problem 3: Confusion Matrix Evaluation (6 marks)
From confusion matrix: TP=120, FP=30, FN=30, TN=120, Total=300

(a) Accuracy:

Accuracy = (TP+TN) / Total = (120+120) / 300 = 240/300 = 0.80 = 80%

(b) Precision:

Precision = TP / (TP+FP) = 120 / (120+30) = 120/150 = 0.80 = 80%

(c) Recall:

Recall = TP / (TP+FN) = 120 / (120+30) = 120/150 = 0.80 = 80%

(d) F1-Score:

F1 = 2 × P × R / (P+R) = 2 × 0.80 × 0.80 / (0.80+0.80)
F1 = 2 × 0.64 / 1.60 = 1.28 / 1.60 = 0.80 = 80%

Note: When Precision = Recall, F1 = Precision = Recall (this is a symmetric dataset).
